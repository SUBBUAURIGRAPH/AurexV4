## Agent Knowledge Base

### Permanent Memories (From Deployments)

**Memory 1: Layer 2 Scaling Trade-offs**
- Optimistic Rollups: Fast, EVM-compatible, longer finality
- ZK Rollups: Shorter finality, more complex, better privacy
- Sidechains: Faster, separate security model
- State Channels: Instant, limited to 2 parties

**Memory 2: Smart Contract Upgrade Patterns**
- Proxy pattern (delegation)
- Eternal storage pattern
- Strategy pattern for modular code
- Staged rollout for major changes

**Memory 3: DeFi Security Lessons**
- Flash loan safeguards: require balance check before+after
- Reentrancy: use checks-effects-interactions or reentrancy guards
- Oracle manipulation: use multiple sources, time-weighted averages
- Front-running: commit-reveal patterns, MEV protection

**Memory 4: Gas Optimization Wins**
- Batch operations: reduce external calls
- Storage layout: pack variables to single slot
- Event indexing: filter events instead of reading state
- Lazy initialization: defer computation to when needed

**Memory 5: Cross-Chain Design Lessons**
- Bridge security: validate signatures from multiple sources
- Liquidity provision: AMM on destination chains
- Wrapped tokens: standardized wrapping/unwrapping
- Governance: control of wrapped asset on destination
