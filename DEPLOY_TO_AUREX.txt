Date: November 1, 2025

DEPLOYMENT CONFIGURATION:
- Frontend: hms.aurex.in (HTTPS)
- Backend API: apihms.aurex.in (HTTPS)
- SSH Access: ssh subbu@hms.aurex.in
- Working Dir: /opt/HMS
- Repository: git@github.com:Aurigraph-DLT-Corp/HMS.git
- Branch: main
- SSL: /etc/letsencrypt/live/aurexcrt1/privkey.pem, fullchain.pem

STEP 1: BUILD DOCKER IMAGE (Local Machine)
============================================
cd /c/subbuworking/HMS
docker build -t hms-trading-app:v2.2.0 -f Dockerfile .
docker images | grep hms-trading-app

STEP 2: DEPLOY TO AUREX (Local Machine)
========================================
# Option A: Using deployment script
chmod +x DEPLOY_AUREX_PRODUCTION.sh
./DEPLOY_AUREX_PRODUCTION.sh

# Option B: Manual SSH deployment
ssh subbu@hms.aurex.in << 'SCRIPT'
cd /opt/HMS
git clone -b main git@github.com:Aurigraph-DLT-Corp/HMS.git /opt/HMS
docker stop $(docker ps -q) 2>/dev/null || true
docker rm $(docker ps -aq) 2>/dev/null || true
docker-compose -f docker-compose.yml up -d
docker ps
SCRIPT

STEP 3: VERIFY DEPLOYMENT
===========================
# Check services
ssh subbu@hms.aurex.in "docker ps"

# Test API health
curl -k https://apihms.aurex.in/api/health

# Test Frontend
curl -k https://hms.aurex.in

# View logs
ssh subbu@hms.aurex.in "docker logs hms-backend -f"

STEP 4: SSL CERTIFICATE VERIFICATION
======================================
ssh subbu@hms.aurex.in "ls -la /etc/letsencrypt/live/aurexcrt1/"
ssh subbu@hms.aurex.in "openssl x509 -in /etc/letsencrypt/live/aurexcrt1/fullchain.pem -text -noout"

STEP 5: DATABASE MIGRATION
===========================
ssh subbu@hms.aurex.in "docker exec hms-backend npm run db:migrate"

TROUBLESHOOTING
===============
1. SSL Certificate Error:
   ssh subbu@hms.aurex.in "sudo certbot renew --force-renewal"

2. Port Already in Use:
   ssh subbu@hms.aurex.in "sudo lsof -i :443"
   ssh subbu@hms.aurex.in "sudo kill -9 <PID>"

3. Database Connection Failed:
   docker logs hms-postgres
   Check DATABASE_URL in docker-compose.yml

4. Out of Memory:
   docker stats
   Add swap or increase server RAM

5. Container Won't Start:
   docker logs hms-backend
   Check environment variables in .env
