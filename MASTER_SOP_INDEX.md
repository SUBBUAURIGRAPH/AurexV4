## 🎯 Common Workflows

### "I'm joining a new project"
1. Find project's PROJECT_SOP.md or PROJECT_MASTER_SOP_TEMPLATE.md
2. Read quick start section
3. Read role-specific section
4. Review code review guidelines
5. Ask team lead questions

**Time**: 30-45 minutes

### "I need to create a new feature"
1. Read SPARC framework section (Master SOP)
2. Follow SPARC Phases (Specification → Pseudocode → Architecture → Refinement → Completion)
3. Use project checklists for each phase
4. Submit for code review using checklist
5. Deploy following project's deployment checklist

**Time**: 3-7 days (varies by feature)

### "I'm starting a new project"
1. Read MASTER_SOP_IMPLEMENTATION_GUIDE.md (Phase 1)
2. Complete preparation checklist
3. Follow Phase 2 (Adoption) steps
4. Copy and customize PROJECT_MASTER_SOP_TEMPLATE.md
5. Begin Phase 3 (Integration)

**Time**: 1-2 weeks for setup

### "I need to audit a project"
1. Choose audit type (quick/complete/deep) from MASTER_SOP_VERIFICATION_CHECKLIST.md
2. Gather project information
3. Complete relevant checklist
4. Score project
5. Create remediation plan
6. Schedule follow-up

**Time**: 5 min (quick) to 3 hours (deep)

### "I'm scaling across multiple projects"
1. Read CROSS_PROJECT_INTEGRATION_GUIDE.md
2. Establish SOP Committee
3. Customize Master SOP for organization
4. Choose distribution method
5. Implement phases (1-4)
6. Track metrics and improvements

**Time**: 3-6 months

## 📞 FAQ

### "Which document should I read first?"
**A**: Start with MASTER_SOP_README.md. It will guide you to the right documents for your role.

### "How do I know if my project is compliant?"
**A**: Use MASTER_SOP_VERIFICATION_CHECKLIST.md. Start with "Quick Verification" (5 minutes).

### "Can I customize Master SOP for my project?"
**A**: Yes! Use PROJECT_MASTER_SOP_TEMPLATE.md to create your project-specific SOP.

### "What if my project uses different tools?"
**A**: Customize the tool selections in PROJECT_MASTER_SOP_TEMPLATE.md. Core practices remain the same.

### "How often should I review these documents?"
**A**: Initially: Read your role section (15-30 min). Then: Reference as needed. Quarterly: Skim updates to main standards.

### "Where do I ask questions about these standards?"
**A**: Ask your team lead, tech lead, or refer to "Questions or Feedback" in MASTER_SOP_README.md.
