## 🚀 Installation Methods

### Method 1: Git Submodule (Recommended for Projects)

Best for: Team projects with automatic updates

```bash
# Add to your project as submodule
cd /your/aurigraph/project
git submodule add git@github.com:Aurigraph-DLT-Corp/glowing-adventure.git .claude
git submodule update --init --recursive

# Start using agents
@devops-engineer "Deploy to dev4"
@jeeves4coder "Review this code"
```

**Benefits**:
- Automatic updates via `git submodule update --remote`
- Version controlled
- Easy synchronization across team
- Zero configuration required

### Method 2: Direct Clone

Best for: Simple setup, local customization

```bash
# Clone into your project
cd /your/aurigraph/project
git clone git@github.com:Aurigraph-DLT-Corp/glowing-adventure.git .claude

# Start using agents immediately
@qa-engineer "Run all tests"
```

**Benefits**:
- Simple one-time setup
- Can modify locally
- No submodule complexity
- Immediate availability

### Method 3: NPM Plugin (Jeeves4Coder)

Best for: Standalone use, CI/CD integration

```bash
# Install from internal registry
npm install @aurigraph/jeeves4coder-plugin --registry https://npm.aurigraph.io

# Or from GitHub
npm install git+https://github.com:Aurigraph-DLT-Corp/glowing-adventure.git#main:plugin

# Use programmatically
const Jeeves4Coder = require('@aurigraph/jeeves4coder-plugin');
const reviewer = new Jeeves4Coder();
```

**Benefits**:
- NPM ecosystem integration
- Programmatic access
- CI/CD pipeline integration
- Dependency management

### Method 4: Download ZIP

Best for: Quick evaluation, no git access

```bash
# Download and extract
wget https://github.com/Aurigraph-DLT-Corp/glowing-adventure/archive/refs/heads/main.zip
unzip main.zip -d .claude

# Start exploring
cat .claude/docs/QUICK_START.md
```

**Benefits**:
- No git required
- Fastest download
- Good for testing
- Portable installation
